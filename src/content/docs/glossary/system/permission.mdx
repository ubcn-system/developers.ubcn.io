## 권한이란?

`권한`은 **계정이 전산에서 접근할 수 있는 대상과 수행할 수 있는 작업 범위를 정의하는 기준**이에요.

유비씨엔 전산에서는 역할 기반 권한과 함께, **단말기 담당자 지정 방식**을 사용해 접근 범위를 제어해요.

</Frame>

<Frame>

## 권한과 계정의 관계

권한은 계정에 부여돼요. 계정의 역할에 따라 기본 권한 범위가 정해지고, 추가 설정에 따라 접근 대상이 제한돼요.

권한이 없는 계정은 전산 기능을 정상적으로 사용할 수 없어요.

</Frame>

<Frame>

## 권한의 적용 기준

권한은 다음 요소를 조합해 적용돼요.<br/><br/>

- 계정 역할 (전체 관리자 / 운영자 / 관리자 유형)
- 상점 단위
- 그룹 구조
- 단말기 담당자 지정 여부

이를 통해 계정별 접근 범위를 세밀하게 제어할 수 있어요.

</Frame>

<Frame>

## 단말기 담당자 기반 접근 제어

운영자 계정은 **담당자로 지정된 단말기만 접근**할 수 있어요.

<br/>

- 담당 단말기: 조회 및 관리 가능
- 비담당 단말기: 접근 불가

이 방식은 권한 설정과 유사한 역할을 하며, 현장 운영에 적합한 제어 방식이에요.

</Frame>

<Frame>

## 그룹 관리자 권한 구조

그룹 관리자는 **지정된 그룹과 그 하위 그룹까지만 관리**할 수 있어요.

<br/>

- 하위 그룹: 접근 가능
- 상위 그룹: 접근 불가

그룹 관리자는 조직 단위 운영을 위한 선택적 관리자예요.

</Frame>

<Frame>

## 권한 관리 원칙

권한은 업무 수행에 필요한 범위까지만 부여하는 것이 좋아요.

정산 정보, 설정 변경과 같은 민감한 기능은 **전체 관리자 계정으로 제한**하는 것을 권장해요.

</Frame>

<Frame>

## 정리

- 권한은 계정의 접근 대상과 작업 범위를 정의
- 역할 기반 + 단말기 담당자 기반 제어를 함께 사용
- 운영자는 담당 단말기만 접근 가능
- 그룹 관리자는 하위 그룹만 관리
- 최소 권한 원칙이 중요

</Frame>
