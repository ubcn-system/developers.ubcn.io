## 로그 / 이력이란?

`로그 / 이력`은 **전산에서 발생한 모든 처리 과정과 변경 사항을 기록한 데이터**를 의미해요.

시스템 동작을 추적하고 문제를 분석하기 위한 중요한 정보예요.

</Frame>

<Frame>

## 로그와 이력의 차이

로그와 이력은 비슷하지만 목적이 달라요.<br/><br/>

- **로그**
  시스템 내부 동작, 통신, 오류 등의 기록

- **이력**
  사용자의 행동이나 데이터 변경 내역 기록

</Frame>

<Frame>

## 기록되는 주요 로그 / 이력

유비씨엔 전산에서는 다음 정보들이 기록돼요.<br/><br/>

- 결제 승인·실패 기록
- API 요청 및 응답
- 상태정보 수신 기록
- 계정 로그인 및 작업 내역
- 설정 변경 이력

</Frame>

<Frame>

## 로그 / 이력의 활용

로그와 이력은 다음 목적에 사용돼요.<br/><br/>

- 장애 원인 분석
- 거래 오류 추적
- 보안 및 접근 기록 확인
- 고객 문의 대응 근거 자료

</Frame>

<Frame>

## 로그 / 이력과 전산 운영

모든 전산 작업은 계정 기준으로 기록돼요.

로그와 이력은 삭제되거나 임의로 수정되지 않으며, **전산 운영의 신뢰성을 유지하는 역할**을 해요.

</Frame>

<Frame>

## 정리

- 로그 / 이력은 전산 활동의 기록
- 장애 분석과 운영 안정성에 필수
- 결제·운영 전반에 걸쳐 활용
- 임의 수정 불가, 신뢰성 유지 수단

</Frame>
