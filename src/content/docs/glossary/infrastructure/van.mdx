## VAN이란?

`VAN`은 **Value Added Network**의 약자로, **단말기에서 발생한 결제 승인 요청을 카드사로 전달하는 통신망 서비스**를 의미해요.

주로 오프라인 결제 환경에서 사용돼요.

</Frame>

<Frame>

## VAN의 역할

VAN사는 다음과 같은 역할을 수행해요.<br/><br/>

- 단말기 승인 요청을 카드사로 전달
- 카드사 승인 결과를 단말기로 전달
- 승인 통신의 안정성과 신뢰성 제공

VAN사는 결제 승인 통신에 특화된 역할을 수행해요.

</Frame>

<Frame>

## VAN과 결제 흐름

VAN 가맹점의 결제 흐름은 다음과 같아요.<br/><br/>

단말기 → 유비씨엔 서버 또는 VAN망 → 카드사 → VAN망 → 단말기

일부 VAN 환경에서는 PG 단계를 거치지 않고 카드사와 직접 통신해요.

</Frame>

<Frame>

## VAN과 정산

VAN 가맹점의 경우, 카드사에서 **직접 상점 계좌로 정산 대금이 입금**되는 경우가 많아요.

VAN사는 정산 대금을 보관하지 않으며, 승인 통신을 중심으로 역할을 수행해요.

</Frame>

<Frame>

## VAN과 PG의 차이

VAN은 승인 통신 중심, PG는 결제 서비스 전반을 대행해요. 유비씨엔 전산에서는 상점의 계약 형태에 따라 PG 또는 VAN 구조를 선택해 사용할 수 있어요.

</Frame>

<Frame>

## 정리

- VAN은 결제 승인 통신망 서비스
- 오프라인 단말기 결제에 특화
- 카드사와 직접 통신 가능
- 정산은 카드사에서 직접 처리하는 경우가 많음

</Frame>
