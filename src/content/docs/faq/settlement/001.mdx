## ❓ 정산 금액이 예상과 다른 이유는 무엇인가요?

정산 금액은 단순히 **승인된 금액의 합계**가 아니에요.
정산 대상 거래는 매입 상태와 취소 여부에 따라 달라질 수 있어요.

다음과 같은 경우에는 정산 금액이 예상과 다르게 입금될 수 있어요.

<br/>

### 1️⃣ 매입보류 또는 매입거절이 발생한 경우

일부 거래가 **매입보류** 또는 **매입거절** 상태인 경우, 해당 금액은 정산 대상에서 제외돼요.

- **매입보류**: 카드사 확인 또는 추가 검증이 필요한 상태
- **매입거절**: 카드사 정책 또는 데이터 오류 등으로 매입이 승인되지 않은 상태

이 경우 해당 거래 금액은 정산에 포함되지 않아요.

<br/>

### 2️⃣ 매입취소가 발생한 경우

이미 정산이 완료된 거래라면 이후 **매입취소**가 발생 시,
정산된 금액만큼 다음 정산에서 **차감되어 입금**돼요.

즉, 정산 금액에서 취소 금액이 빠진 상태로 지급돼요.

<br/>

### 3️⃣ 승인과 정산은 다른 단계입니다

승인이 성공했다고 해서 반드시 정산까지 완료되는 것은 아니에요.

정산은 다음과 같은 흐름을 거쳐 확정돼요.

```

승인 → 구매 → 마감 → 매입 → 정산

```

각 단계에서 상태가 변경될 수 있기 때문에, 최종 정산 금액은 승인 금액과 차이가 발생할 수 있어요.

<br/>

## ✔ 정리

정산 금액은 다음 요소를 반영해 계산돼요.

- 매입 완료된 거래 금액
- 매입보류·매입거절 거래 제외
- 매입취소 금액 차감

정확한 확인을 위해서는 **거래 상태와 매입 상태를 함께 확인**하는 것이 좋아요.
